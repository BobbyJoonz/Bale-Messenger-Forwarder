# یوزربات انتقال پیام بله

این پروژه با یک **اکانت شخصی بله** وارد میشود و پیامهای جدید **یک یا چند چت مبدا** را به یک چت مقصد منتقل میکند.

## قابلیتها

- ورود با شماره تلفن، کد تایید و در صورت نیاز رمز دومرحلهای
- ذخیره سشن؛ در اجرای بعدی دوباره کد نمیخواهد
- پشتیبانی از **چندین چت مبدا** همزمان
- پشتیبانی از چت خصوصی، گروه، سوپرگروه و کانال
- دو حالت انتقال:
  - `forward`: فوروارد واقعی پیام
  - `copy`: متن را بدون برچسب فوروارد کپی میکند و فایل/رسانه را با محتوای موجود دوباره میفرستد
- فیلتر اختیاری `allowed_sender_id` برای انتقال پیامهای فقط یک کاربر
- جلوگیری دائمی از ارسال تکراری با SQLite
- تلاش مجدد خودکار و اتصال مجدد کتابخانه
- حالت `--inspect` برای پیدا کردن `chat_id`، نوع چت و `sender_id`
- رفع باگ BaleClient مربوط به string annotations

## اجرای سریع در ویندوز

1. Python نسخه 3.11 یا جدیدتر را نصب کنید و گزینه **Add Python to PATH** را فعال کنید.
2. فایل ZIP را Extract کنید.
3. برای پیدا کردن شناسهها، در CMD داخل پوشه اجرا کنید:

```bat
run_windows.bat --inspect
```

4. برنامه از شما شماره بله را بهصورت بینالمللی میخواهد؛ برای ایران مثل `989121234567` و بدون علامت `+` وارد کنید.
5. داخل چت مبدا و مقصد یک پیام بفرستید. در پنجره CMD مقادیر زیر چاپ میشوند:

```text
chat_id=...
chat_type=PRIVATE/GROUP/CHANNEL/SUPER_GROUP
sender_id=...
```

6. با `Ctrl+C` برنامه را ببندید و تنظیمات را بسازید:

```bat
run_windows.bat --setup
```

7. دفعات بعد فقط این فایل را اجرا کنید:

```bat
run_windows.bat
```

## اجرای لینوکس یا VPS

```bash
chmod +x run_linux.sh
./run_linux.sh --inspect
./run_linux.sh --setup
./run_linux.sh
```

## اجرا به عنوان سرویس systemd (لینوکس)

برای اینکه برنامه بعد از ریستارت سرور خودکار بالا بیاد:

```bash
sudo cp bale-relay.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable bale-relay
sudo systemctl start bale-relay

# مشاهده لاگ
journalctl -u bale-relay -f
```

## معنی تنظیمات

فایل `config.json` پس از Setup ساخته میشود. نمونه: `config.example.json`

### فرمت جدید (چند مبدا):

```json
{
  "sources": [
    { "id": 111111111, "type": "PRIVATE" },
    { "id": 222222222, "type": "CHANNEL" }
  ],
  "target": { "id": 999999999, "type": "PRIVATE" },
  "mode": "forward"
}
```

### فرمت قدیمی (تک مبدا - همچنان پشتیبانی میشود):

```json
{
  "source": { "id": 111111111, "type": "PRIVATE" },
  "target": { "id": 999999999, "type": "PRIVATE" },
  "mode": "forward"
}
```

### تنظیمات:

| کلید | توضیح |
|---|---|
| `sources` | لیست چتهای مبدا (آرایه‌ای از `{id, type}`) |
| `source` | چت مبدا تکی (فرمت قدیمی، همچنان کار میکند) |
| `target` | شناسه و نوع چت مقصد |
| `mode` | `forward` یا `copy` |
| `allowed_sender_id` | اگر `null` باشد همه پیامها منتقل میشوند؛ اگر عدد باشد فقط پیامهای آن فرستنده |
| `mark_as_read` | بعد از انتقال موفق، چت مبدا خواندهشده علامت بخورد |
| `copy_fallback_to_forward` | در حالت copy، انواع ناشناخته فوروارد شوند |
| `delay_seconds` | فاصله میان انتقالها (پیشفرض: 0.35) |
| `max_retries` | تعداد تلاش مجدد هنگام خطا (پیشفرض: 4) |
| `retry_base_seconds` | مبنای تاخیر تصاعدی (پیشفرض: 1.5) |
| `dedupe_max_rows` | حداکثر سابقه ذخیرهشده (پیشفرض: 20000) |

## ورود با شمارهای دیگر

برای حذف سشن فعلی و ورود مجدد:

```bat
run_windows.bat --reset-session
```

## فایلهای مهم

| فایل | توضیح |
|---|---|
| `account_session.bale` | سشن ورود — **بسیار محرمانه** — هرگز به اشتراک نگذارید |
| `config.json` | شناسهها و تنظیمات انتقال |
| `relay_state.sqlite3` | سابقه پیامهای منتقلشده |
| `relay.log` | گزارش اجرا و خطاها |

## نکات مهم

- اکانت واردشده باید به **همه چتهای مبدا و مقصد** دسترسی داشته باشد
- در مقصد باید اجازه ارسال پیام داشته باشد
- مبدا و مقصد نباید یکسان باشند
- از API داخلی و غیررسمی بله استفاده میکند؛ تغییرات بله ممکن است نیازمند بروزرسانی باشد
- فقط روی اکانت و چتهایی استفاده کنید که مالک آنها هستید یا اجازه صریح دارید
